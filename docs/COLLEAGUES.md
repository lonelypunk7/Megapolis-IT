# Гайд для коллег — Работа с TestY через Qwen Code

## Что это

Вы общаетесь с AI-ассистентом (Qwen Code) прямо в VS Code. Он работает с TestY TMS через API и может:
- Создавать тест-кейсы
- Редактировать/удалять кейсы
- Искать кейсы по проектам/сьютам
- Создавать сьюты, тест-планы, результаты
- Работать с комментариями, пользователями, лейблами

## Как начать

### 1. Откройте проект

Откройте VS Code → `File → Open Folder` → выберите папку `/tmp/Megapolis-IT`

### 2. Убедитесь, что `.env` есть

```bash
# В терминале (или проверьте что файл есть)
ls .env
```

Если нет — скопируйте:
```bash
cp .env.example .env
```

И впишите свои креды в `.env`:
```
TESTY_LOGIN=ваш_логин
TESTY_PASSWORD=ваш_пароль
```

### 3. Откройте чат Qwen Code

- `Ctrl+Shift+P` → `Qwen Code: Chat`
- Или просто откройте боковую панель Qwen Code и пишите в чат

### 4. Начните работать!

Пишите на русском. Вот примеры команд:

## Правила тест-кейсов

Смотри `testy/case_rules.md` — полный гайд.

**Кратко:**
- `is_steps: true` всегда
- Не более 5 шагов на кейс
- Каждый шаг — одно действие
- Конкретные названия, не "что-то"
- `setup` — состояние системы, не действия
- `description` начинается с "Цель: проверить, что ..."

## Частые вопросы

### Q: Как создать тест-кейс?
**A:** Напишите в чат: "Создай тест-кейс в Smoke на проверку X. Описание: ..."

### Q: Как найти кейс?
**A:** "Покажи все кейсы в проекте 22" или "Найди кейсы по запросу 'авторизация'"

### Q: Как создать сьют?
**A:** "Создай сьюту 'Название' в проекте 22"

### Q: Как добавить комментарий?
**A:** "Добавь комментарий к кейсу 4557: 'Текст комментария'"

### Q: Как удалить кейс?
**A:** "Удали кейс 4556"

### Q: Что если что-то не работает?
**A:** Напишите: "Проверь авторизацию в TestY" — ассистент проверит подключение.
